## Components

### AppFrame
* **V4 Behavior:** Renders dropdowns in a portal, outside of the main DOM tree.
* **V3 Behavior:** Renders dropdowns in DOM with header bar.

### Button
* **V4 Behavior:** All buttons, including those with the 'add' icon, will have margin on their right side unless other styles from, for instance, a container remove it.
* **V3 Behavior:** Buttons with the icon "add" have their margin removed.
* **Recommendation:** Set the `noSpacing` prop to true on a button to keep the margin removed.

### ButtonBar
* **V4 Behavior:** The discard button will be styled the same as a cancel button.
* **V3 Behavior:** The discard button is styled like a standard button.
* **V4 Behavior:** Renders button tooltips in a portal, outside of the main DOM tree.
* **V3 Behavior:** Renders button tooltips in DOM with button bar.

### Calendar
* **V4 Behavior:** Renders calendar in a portal, outside of the main DOM tree.
* **V3 Behavior:** Renders calendar in DOM with input.
* **V4 Behavior:** When provided, the `date` prop will control the value of the component.
* **V3 Behavior:** The `date` prop only updates the component when it changes.

### ColorPicker
* **V4 Behavior:** Renders color picker in a portal, outside of the main DOM tree.
* **V3 Behavior:** Renders color picker in DOM with input.
* **V4 Behavior:** The component controls the state of `color` and `open` if they're not defined as props. The initial values can be set with `initialState.value`. Also, an internal error message is managed for invalid hex values.
* **V3 Behavior:** The `stateless` prop determines whether `open` and `errorMessage` are controlled by the component or not.

### ConditionalFieldset
* **V4 Behavior:** Renders dropdown in a portal, outside of the main DOM tree.
* **V3 Behavior:** Renders dropdown in DOM with input.
* **V4 Behavior:** When provided, the `selectedIndex` prop will control which index is selected. Otherwise, the component will handle which index is selected internally.
* **V3 Behavior:** The `stateless` prop decides whether or not the index state will be controlled by the component.

### ConfirmTooltip
* **V4 Behavior:** Renders in a portal, outside of the main DOM tree.
* **V3 Behavior:** Renders in DOM with trigger.

### Country Selector
* **V4 Behavior:** Renders dropdown in a portal, outside of the main DOM tree.

* **V4 Behavior:** The component controls the state of `open`, `searchIndex`, `searchString`, and `searchTime` if they're not defined as props. The initial values can be set with `initialState`.

### DetailsTooltip
* **V4 Behavior:** Renders in a portal, outside of the main DOM tree.
* **V3 Behavior:** Renders in DOM with trigger.
* **V4 Behavior:** When provided, the `open` prop will control whether the component is open. Otherwise, the component will handle open and closing internally.
* **V3 Behavior:** The `stateless` prop decides whether or not the open state will be controlled by the component.
* **V4 Behavior:** `placement` prop accepts "top", "bottom", "top left", "top right", "bottom left", and "bottom right" to position.
* **V3 Behavior:** `positionClassName` is added to classes of component. CSS does the positioning.
* **Recommendation:** Use `placement` instead of `positionClassName`.
* **V4 Behavior:** The alert style of the DetailsTooltip will be triggered only by setting `type` to "alert".
* **V3 Behavior:** Adding "alert" to `positionClassName` triggers alert styling.
* **Recommendation:** Use `type` = "alert" instead of `positionClassName`.

### ExpandableRow
* **V4 Behavior:** Renders dropdown in a portal, outside of the main DOM tree.
* **V3 Behavior:** Renders dropdown in DOM with row.
* **V4 Behavior:** Each ExpandableRow will use a standalone CSS class to style them even out of context.
* **V3 Behavior:** ExpandableRows get their styling from a surrounding element.
* **V4 Behavior:** When provided, the `expanded` prop will control whether the component is open. Otherwise, the component will handle open and closing internally.
* **V3 Behavior:** The `stateless` prop decides whether or not the open state will be controlled by the component.
* **V4 Behavior:** The `onToggle` callback behaves consistently and returns the "before" state.
* **V3 Behavior:** The stateful version calls `onToggle` with the new state of `expanded` while the stateless version calls it with the old state.
* **V4 Behavior:** When provided, the `showDeleteConfirm` prop will control whether the confirm delete tooltip is shown. Otherwise, the component will the tooltip.
* **V3 Behavior:** The `stateless` prop decides whether or not the confirm delete tooltip state will be controlled by the component.

### FileUpload
* **V4 Behavior:** When provided, the `defaultImage` prop will show an image with opacity when no `thumbnailSrc` prop is provided.
* **V3 Behavior:** The `defaultImage` prop worked roughly the same as `thumbnailSrc` and took precedence over it, but only in the stateful version of the component. There was no additional styling.

### HeaderBar

### Icon
* **V4 Behavior:** Displays an icon with only a SPAN tag unless it contains `children` or `type` is set to "leading".
* **V3 Behavior:** Displays a block element with an icon that has right margin unless `type` is set to "inline".
* **Recommendation:** Set `type` to "leading" for block element icons.

### LeftNavBar

### Link
* **V4 Behavior:** Displays a link with only an A tag unless `type` is set to "block".
* **V3 Behavior:** Displays a block element with a link inside it unless it:
    * has `children` or some `type` other than block is set
    * doesn't have `title` or `count` set
