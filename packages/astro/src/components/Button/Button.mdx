# Button

Button is used primarily to trigger an action or event. It is built on top of
[Button from Theme-UI](https://theme-ui.com/components/button). The stylings of a button is different based on its purpose.
(Primary Action, Secondary Action, Danger Button)

### Required components 

- Button component can be used independently and does not require additional components.

### Best practices

Button should:
- be clearly and accurately labeled.

- be used as links.
- override colors (accessibility purposes).
- use uppercase titles.

### Accessibility

This component should adhere to the [WAI-ARIA Button](https://www.w3.org/TR/wai-aria-practices-1.1/examples/button/button.html) accessibility guidelines.

#### Keyboard Navigation

- Buttons are focusable using the Tab key and can be selected using the Space or Enter key. 

#### Screen readers.

Button utilizes `aria-label` to give an accessible name to the component.

| Aria Attribute | Role | Component |
| ---- | ---- | ----------- |
| `aria-label` | button | Button |
